# AI Screen Code - 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品定义

**产品名称**: AI Screen Code
**产品类型**: Web应用 + 桌面应用 (Tauri)
**核心价值主张**: 将UI设计图片自动转换为可用的前端代码，降低开发成本，提升开发效率
**目标用户**:
- 前端开发者 (主要) - 快速生成组件代码
- UI设计师 (次要) - 验证设计可实现性
- 产品经理 (辅助) - 快速原型验证

## 2. 用户交互流程

### 2.1 主流程 (Happy Path)

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           用户主流程                                         │
└─────────────────────────────────────────────────────────────────────────────┘

[首页] ──> [上传图片] ──> [选择模型/语言] ──> [生成代码] ──> [预览/复制/下载]
    │                                                              │
    └──────────────────────────────────────────────────────────────┘
                                │
                    [查看历史记录] ◄─────────────────┘
```

### 2.2 详细步骤说明

#### 步骤1: 首页 (Home Page)
1. 用户访问首页，查看产品介绍
2. 点击上传区域或拖拽图片到上传区域
3. 选择图片后显示预览缩略图
4. 选择目标AI模型 (默认: Qwen)
5. 选择输出语言 (默认: React)
6. 点击"生成代码"按钮跳转到生成页面

#### 步骤2: 代码生成 (Generate Page)
1. 自动开始生成流程
2. 显示进度: 上传 → 分析 → 生成 → 完成
3. 错误处理: API Key无效/网络超时
4. 成功后显示代码编辑器

#### 步骤3: 代码操作
1. 复制代码到剪贴板
2. 下载代码到本地文件
3. 查看实时预览效果 (Web语言)
4. 返回首页继续生成

### 2.3 次要流程

#### 设置流程
1. 访问设置页面
2. 输入各模型API Key
3. 点击验证按钮确认有效性
4. 配置自定义Base URL (可选)

#### 历史记录流程
1. 访问历史记录页面
2. 浏览历史生成记录卡片
3. 点击查看详情
4. 重新生成或删除记录

## 4. 数据模型设计

### 4.1 核心数据模型

#### 用户 (User)
```typescript
interface User {
  id: string;              // UUID
  username: string;        // 用户名
  email: string;           // 邮箱
  nickname?: string;        // 昵称
  avatar?: string;         // 头像URL
  created_at: string;      // 创建时间
  role: 'free' | 'pro';   // 用户角色
}
```

#### 生成记录 (History)
```typescript
interface History {
  id: string;              // UUID
  user_id?: string;        // 用户ID (可选，未登录时为空)
  image_base64: string;    // 图片Base64
  image_url?: string;      // 图片URL (可选)
  code: string;           // 生成的代码
  language: string;        // 语言类型
  model: string;           // 使用的模型
  model_response?: string; // 模型的原始响应
  prompt?: string;        // 使用的提示词
  generation_time_ms?: number; // 生成耗时(毫秒)
  created_at: string;     // 创建时间
}
```

#### 用户配置 (UserConfig)
```typescript
interface UserConfig {
  user_id?: string;        // 用户ID
  default_model: AIModel; // 默认模型
  default_language: ProgrammingLanguage; // 默认语言
  api_keys: Record<AIModel, string>; // 各模型API Key
  custom_base_urls: Record<AIModel, string>; // 自定义端点
  theme: 'light' | 'dark'; // 主题
  auto_save: boolean;     // 自动保存到历史
}
```

#### API Key验证记录 (ApiKeyValidation)
```typescript
interface ApiKeyValidation {
  id: string;
  model: AIModel;
  valid: boolean;
  message: string;
  validated_at: string;
}
```

### 4.2 枚举类型

```typescript
// AI模型
type AIModel = 'qwen' | 'minimax' | 'kimi' | 'glm';

// 编程语言
type ProgrammingLanguage = 'react' | 'vue' | 'kotlin' | 'swift';

// 生成状态
type GenerationStatus = 'pending' | 'processing' | 'completed' | 'failed';

// 主题
type Theme = 'light' | 'dark' | 'system';
```

### 4.3 API响应格式

```typescript
// 通用响应
interface ApiResponse<T> {
  success: boolean;
  data?: T;
  error?: string;
  message?: string;
}

// 分页响应
interface PaginatedResponse<T> {
  success: boolean;
  data: T[];
  meta: {
    total: number;
    page: number;
    limit: number;
    has_more: boolean;
  };
}

// 生成响应
interface GenerateResponse {
  id: string;
  code: string;
  language: string;
  preview_url?: string;
  generation_time_ms: number;
}
```

## 5. 错误处理设计

### 5.1 错误类型

| 错误码 | 错误类型 | 用户提示 | 处理方式 |
|--------|---------|---------|---------|
| 4001 | 无效的图片格式 | "只支持 PNG、JPG、WebP 格式" | 提示正确格式 |
| 4002 | 图片太大 | "图片大小不能超过 10MB" | 建议压缩 |
| 4003 | 无效的API Key | "API Key 无效，请检查配置" | 跳转设置 |
| 4004 | API调用超时 | "生成超时，请稍后重试" | 重试按钮 |
| 4005 | 余额不足 | "API余额不足，请充值" | 跳转充值 |
| 5001 | 服务器错误 | "服务器繁忙，请稍后重试" | 重试/联系客服 |

### 5.2 前端错误展示

```typescript
// Toast 提示
toast.error('API Key 无效，请检查配置', {
  action: {
    label: '前往设置',
    onClick: () => navigate('/settings'),
  },
});

// 页面级错误
<ErrorBoundary fallback={<ErrorPage />}>
  <Component />
</ErrorBoundary>
```

## 6. 性能要求

|------|-------|------|

|------|-------|------|
| 首次响应 | < 5s | 流式输出首字节 |

| 并发限制 | 3个 | 同一用户 |

## 7. 安全设计

### 7.1 数据安全

- API Key存储在浏览器localStorage (仅本地)
- 图片Base64仅在内存中处理，不持久化
- 生成的代码不包含用户敏感信息

### 7.2 接口安全

- CORS配置限制来源
- 请求频率限制 (Rate Limiting)
- 输入验证和过滤
- SQL注入防护
